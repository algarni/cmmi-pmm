Main Description

Any number of changes can be made to a CI before baseline is established. CI’s are tagged/labeled on meeting baseline criteria as specified in the CM plan. Each CI’s tag/label is related to labels of predecessors and siblings. This relationship must be made very obvious. Once a baseline is created, if a change is made, it results in a new baseline. Once a baseline is established for a CI, changes can be made in a controlled manner by adhering to the change control process. This provides for tracking the changes made to a CI and if required the CI can be rolled back to undo a change.

 

Baseline criteria for all Configuration Items are identified as part of the Software Configuration Management Plan. On completion of baseline criteria, the Configuration Item shall be tagged / labeled by the Configuration Controller with the baseline identification in accordance with the Software Configuration Management Policy.
