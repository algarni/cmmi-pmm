Task Descriptor: Establish Project’s Environment and CM Plan
This task prepares the PMM lifecycle management environment for the project, where the project environment includes both process and tools and establishes the Configuration Management Plan for the project to guide the implementation of Configuration Management activities specific to the engagement.

Main Description

The Configuration Management plan is prepared by the Configuration Controller while planning the software development process improvement activities. The configuration management plan defines:

·       Identification of Configuration Items

·       Naming convention for CIs, Storage Location of the Cis (Server/Folder name) and Baseline Criteria of CIs

·       Version Numbering of Cis.

The CM plan evolves as the project progresses. During the project execution it may be required to add new configuration items, modify baseline criteria and modify baseline labeling or product versioning conventions. The revised CM plan should be shared with all the affected groups of the project and their commitment obtained.
