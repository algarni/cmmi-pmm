Task Descriptor: Perform Configuration Audit
This task is designed to perform configuration audits for assisting the project team and senior management confirming that the resulting baselines from configuration management conform to the specified standards and plans.

Main Description

Configuration audits shall be conducted by QA Engineer along with Configuration Controller to ensure that what has been designed has in fact been built and the tests applied to a CI prove that requirements for a CI have been satisfied. Configuration audits are required to be conducted for every phase before exiting the phase.

 

At RSLF the following types of configuration audits are performed:

1.         Functional Configuration Audit (FCA): A functional configuration audit verifies that the configuration item (CI) has been completed satisfactorily, that the item has achieved the specified performance and functional characteristics, and that its operational and support documents are complete and satisfactory.   During this audit, CM will analyze the projectâ€™s requirements traceability matrix, requirements documentation, test reports, problem reports, change authorizations, and any waivers or deviations.

2.         Physical Configuration Audit (PCA): A physical configuration audit verifies that a configuration item (CI), as built, conforms to the technical documentation that defines and describes it (i.e., it verifies that all baseline elements of the delivery match the documentation).

At RSLF, the configuration audit is a combination of FCA and PCA configuration audit types. The issues identified during configuration audit are recorded in the Configuration Audit Checklist and tracked to closure.
