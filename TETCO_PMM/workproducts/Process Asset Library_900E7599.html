Brief Outline

The structure of the NIC Application Development Unified Process (ADUP) Process Assets Library (PAL) can be illustrated as follows:

 

Process Assets Library (PAL) Views

 

Team View

This view comprises the basic elements and method content of the NIC ADUP including:

 

1.        Roles

Describe the NIC ADUP fundamental roles that are used by Tasks to specify who performs them as well as define a set of Work Products they are responsible for.

 

2.      Work Products

Collection of ADUP Work Product Domains and Work Product Kinds.

 

3.      Templates

Specify the structure of a Work Product by providing a pre-defined table of contents, sections, packages, and/or headings, a standardized format, as well as descriptions how the sections and packages are supposed to be used and completed.

 

4.      Disciplines

This section describes the NIC ADUP depicts structure of the NIC ADUP disciplines.

 

5.      Measures and Reports

This guideline describes a set of standard SMO software measures that are compatible with the measurement practices described in the Capability Maturity Model for Software.

 

6.      Policies

This section describes the NIC ADUP polices designed to guide the implementation of the NIC ADUP in a software project.

 

7.       Checklists

Checklists are lists of items that need to be completed or verified. Checklists are used in reviews including project management reviews, technical peer reviews and quality assurance audits.

 

8.      Tool Mentors

 Tool Mentors explain how to use a specific tool to create part of a Work Product either in the context of or independently from a Task or Activity. All tool mentors are gathered here and organized by tool.

 

9.      CMMI Mapping

This view includes the mapping between the CMMI v 1.3 process areas specific and generic practices and the NIC ADUP method content (e.g. tasks, work products, roles, guidance)

 

Lifecycle View

This view includes the Delivery Processes and lifecycle models published in this configuration and used to deliver NIC software products including the following:

 

1.        NIC ADUP Typical Lifecycle

The purpose of this Process is to provide comprehensive representation of the NIC software typical lifecycle phases for non-maintenance projects.

 

2.      NIC ADUP Maintenance Lifecycle

The purpose of this Process is to provide comprehensive representation of the NIC software maintenance lifecycle the maintenance change requests.
